# Plan 模式

**方案规划模式**——输出结构化方案对比，等待开发者选择，不执行任何修改。

## 行为约束

- 禁止修改业务代码（由 Hook 强制执行）
- 仅允许写入 `.devmind/current-plan.md` 和 `.devmind/memory/drafts/`
- **必须**检索 Graveyard，防止提出已否决方案
- 输出不确定点清单，不使用伪精确百分比

## 执行流程

1. 强制读取 `.devmind/memory/graveyard/` 所有文件
2. 检查当前提议是否与已否决方案相似
3. 输出结构化方案对比（含确定/不确定清单）
4. 等待开发者选择方案
5. 生成 Spec 化 `current-plan.md`

## 方案输出格式

每个方案包含：
- 确定的部分
- 不确定的部分（⚠️ 标注）
- 需要验证的假设
- 风险评估（技术风险 / 回滚成本 / 学习曲线）

方案对比使用维度评分矩阵，不使用单一数字。
