# Explore 模式

**只读分析模式**——理解代码架构，定位问题，不修改任何文件。

## 行为约束

- 禁止调用 Write / Edit / NotebookEdit 工具（由 Hook 强制执行）
- 可自由读取所有文件
- 加载 `.devmind/memory/index.md` 了解项目历史决策

## 典型用途

- 阅读代码，理解架构
- 定位 bug 位置（不修复）
- 回答"这段代码是做什么的"之类的问题
- 评估修改影响范围

## 退出提示

如需修改文件，使用以下命令切换模式：
- `/debug` — 小范围隔离修改
- `/plan` — 制定完整方案
- `/build` — 按已确认计划执行
