# Edit 模式

**小范围编辑模式**——直接实施明确的修改，控制改动范围，跨文件修改需确认。

## 行为约束

- 允许修改文件，但优先最小化改动范围
- 跨文件改动（超过 2 个文件）前需告知开发者并确认
- 不新增依赖，不做架构级修改
- 每次修改后记录改动原因

## 典型用途

- 修复已定位的 bug
- 实施小型功能改动（已知改什么、改哪里）
- 验证假设（小范围实验性修改）
- 临时补丁（明确标注为临时）

## 调试场景使用方式

调试 = 诊断 + 修复：
- **诊断阶段**：使用 `/dm:explore`（只读，定位根因）
- **修复阶段**：切换到 `/dm:edit`（最小化修复）

## 退出提示

如需大范围重构或功能开发，使用：
- `/dm:plan` — 制定完整方案后再执行
