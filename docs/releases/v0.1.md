# Release v0.1

> 发布日期：2026-02-25

## 本版本包含

- **[4 种工作模式系统](../designs/v0.1/work-modes.md)**：定义 explore / plan / build / edit 四种互斥工作模式，由 Hook 在工具调用层强制执行权限边界，彻底解决 AI "越权修改文件"的问题。

- **[Hook 系统：模式约束与审计日志](../designs/v0.1/hook-system.md)**：PreToolUse Hook 在每次写操作前拦截并按当前模式决策放行或阻断；PostToolUse Hook 自动记录所有写操作到审计日志，纯 bash 实现，无外部依赖。

- **[记忆系统：remember / recall / bury / sync-memory](../designs/v0.1/memory-system.md)**：三层本地 Markdown 记忆目录（decisions / patterns / graveyard），轻量 index.md 索引供会话启动加载，check-graveyard.py 在 Plan 模式强制检测已否决方案，防止 AI 重复提议同类路径。

- **[Phase 2 CLI 子命令：init / status / recall / audit](../designs/v0.1/cli-commands.md)**：`devmind` CLI 工具，提供 AI 会话之外的观察入口；walk-up 策略自动定位 `.devmind/` 目录；`devmind init` 幂等初始化项目结构；已发布至 npm（`@lich0821/devmind`）。

- **[publish/release 文档工作流](../designs/v0.1/publish-release-workflow.md)**：`/dm:publish` 将功能整理为人类可读设计文档存入 `docs/designs/draft/`；`/dm:release` 汇总 draft/ 生成版本文档并归档，`draft/` 目录即版本边界，零配置、操作原子。

## 架构变化

v0.1 建立了 DevMind 的核心架构：

```
.devmind/               ← AI 工作空间（模式状态、记忆、计划、会话）
.claude/hooks/          ← 工具调用拦截层（模式约束 + 审计日志）
.claude/commands/dm/    ← 11 条 Slash Commands（工作流驱动）
.claude/CLAUDE.md       ← 会话启动注入（状态感知提示）
packages/cli/           ← devmind CLI（AI 会话外的观察工具）
docs/designs/           ← 人类可读设计文档（AI 生成，人类归档）
```

**关键设计原则**：约束在 Hook 层强制执行，而非依赖 AI 提示词自律；`.devmind/` 与 `docs/` 严格分层，分别服务 AI 工作空间和人类文档需求。

## 已知问题 / 后续计划

**当前限制：**
- `devmind init` 为非交互式，所有配置使用默认值，需手动编辑 `config.yaml` 自定义
- Hook 拦截范围不含 Bash 工具调用（`rm`、`git` 等写操作不受模式约束）
- `devmind recall` 和 `/dm:recall` 为关键词字符串匹配，不支持语义相似度
- `edit` 模式的跨文件确认在 Hook 层未实现（依赖 AI 提示词约束）

**v0.2 计划方向：**
- 交互式 `devmind init` 向导（solo/team 配置问答）
- Phase 3：本地 embedding 语义搜索（embed-local / embed-ollama）
- `devmind mode <mode>` CLI 命令，终端直接切换模式
- `devmind migrate` 工具，支持旧版配置迁移
