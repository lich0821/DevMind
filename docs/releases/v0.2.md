# Release v0.2

> 发布日期：2026-02-26

## 本版本包含

- **[添加 migrate 命令](../designs/v0.2/migrate-command.md)**：新增 `devmind migrate` CLI 命令和 `/dm:migrate` slash command，支持将存量项目（含 OpenSpec 项目）的已有知识迁移到 DevMind 记忆系统

## 架构变化

新增两层迁移机制：

- **CLI 层**（`devmind migrate [path]`）：自动检测技术栈、OpenSpec 结构、git log，生成 `.devmind/migrate-checklist.md`，无需交互
- **Slash Command 层**（`/dm:migrate`）：Claude  Code 读取 checklist，探索代码库，将架构决策写入 `memory/decisions/`、开发规律写入 `memory/patterns/`，OpenSpec 项目还可将进行中的变更转换为 `current-plan.md`

## 已知问题 / 后续计划

- `detectTechStack()` 仅扫描顶层 `package.json`，monorepo 子包的技术栈不被识别
- `/dm:migrate` 对大型项目的代码探索深度有限，可能遗漏深层架构决策（属于设计取舍：宁可少写、写准）
- 后续可考虑：`devmind migrate --dry-run` 选项，只输出 checklist 预览不写文件
