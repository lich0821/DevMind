# Release v0.3

> 发布日期：2026-02-26

## 本版本包含

- **[/dm:auto — 全自动流水线模式](../designs/v0.3/auto-pipeline-mode.md)**：新增 `/dm:auto` 命令，输入一句话需求，自动串联 explore → plan → build 全流程。低风险改动（≤5 个文件、无新依赖、不涉及基础设施）全程无需确认；高风险改动触发一次门控暂停，等待确认后继续。

## 架构变化

`/dm:auto` 作为纯 slash command 实现（`.claude/commands/dm/auto.md`），不新增持久模式标识符。
执行期间复用现有 `explore` 和 `build` 两个模式，Hook 安全约束完全保留。

风险评估以**文件数量**和**依赖变化**作为代理指标，在自动化程度与安全性之间取得平衡。

## 已知问题 / 后续计划

- 风险门控为粗粒度（文件数量），部分单文件高风险改动可能被低估，后续可引入文件类型权重
- Auto 模式 build 阶段暂停后通过 `/dm:build` 续传，尚无专用的 resume 语法
- 后续考虑支持 `/dm:auto --dry-run`，只输出计划不执行
